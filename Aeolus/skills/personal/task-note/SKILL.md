# 任务记录 / 查询 / 状态更新

参数：**$ARGUMENTS**

**第一步：** 运行 `echo $AEOLUS_DIR` 获取 Aeolus 路径，记为 `<AEOLUS_DIR>`。

任务数据文件路径：`<AEOLUS_DIR>/skills/personal/task-note/data/tasks.md`

---

## 判断模式

分析 `$ARGUMENTS`：

- 如果参数**包含任务编号**且包含"完成"、"done"、"finish"等词 → **完成模式**
- 如果参数**包含任务编号**且包含"备注"、"note"、"加备注"等词 → **备注模式**
- 如果参数**包含任务编号** → **记录模式**
- 如果参数为空、或包含"待办"、"todo"、"任务列表"、"我的任务"等词 → **列表模式**
- 否则 → **查询模式**

---

## 文件格式说明

文件为 Markdown 表格，表头：
```
| 状态 | 编号 | 概述 | 备注 |
| --- | --- | --- | --- |
```

每行格式：
```
| ⬜ | BI-XXXXXX | 任务概述 | 备注内容（可为空） |
| ✅ | BI-XXXXXX | 任务概述 | 备注内容（可为空） |
```

- `⬜` 表示未完成，`✅` 表示已完成
- **未完成的任务排在最上面，已完成的排在下面**

---

## 记录模式

1. 读取任务文件（不存在则从空表格开始）
2. 从参数中提取任务编号和概述
3. 新任务默认状态 `⬜`，备注为空
4. 重新排列：未完成在上，已完成在下，写入文件
5. 回复：已记录 `编号` — 概述

---

## 备注模式

1. 读取任务文件
2. 找到对应任务行，更新备注内容
3. 保持状态不变，重新排列写入文件
4. 回复：已更新 `编号` 备注

---

## 完成模式

1. 读取任务文件
2. 找到对应任务行，将状态改为 `✅`
3. 重新排列：未完成在上，已完成在下，写入文件
4. 回复：已标记 `编号` 完成

---

## 列表模式

1. 读取任务文件
2. 分两组展示：
   - **未完成** ⬜：编号 + 概述 + 备注（如有）
   - **已完成** ✅：编号 + 概述
3. 格式简洁，未完成在上

---

## 查询模式

1. 读取任务文件
2. 模糊匹配概述/备注内容
3. 返回匹配的任务行
4. 未找到则告知用户

---

完成后简洁回复，不要啰嗦。
